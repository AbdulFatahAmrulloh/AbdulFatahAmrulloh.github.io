This project showcases a **mobile live streaming application** developed for Android devices using the **StreamPack library**. The system enables real-time video streaming with **low latency**, making it suitable for broadcasting environments such as media production, live reporting, and internal studio monitoring.

The application has been **successfully implemented and used in a real-world production environment**, demonstrating its reliability and performance.

---

## 📱 Application Overview

<img src="/images/preview_main.png" alt="Live Streaming Preview" style="width:100%; border-radius:10px; margin-bottom:20px;">

This app allows users to capture video directly from an Android device and stream it to a remote server using the **SRT (Secure Reliable Transport)** protocol.

---

## 🚀 Key Features

### 🎥 Real-Time Video Streaming
- Live video capture directly from Android camera  
- Smooth streaming with low latency  
- Optimized for stable transmission in local or network environments  

<img src="/images/preview_streaming.jpg" alt="Streaming Activity" style="width:100%; border-radius:10px; margin:20px 0;">

---

### ⚙️ Advanced Encoder Configuration
- Video Encoder: H.264 / AVC  
- Resolution: Up to 1280x720 (HD)  
- Frame Rate: 30 FPS  
- Adjustable Bitrate for network optimization

### 🔊 Audio Streaming Support
- AAC Audio Encoder  
- Stereo Channel  
- Configurable bitrate (e.g., 128 kbps)  
- 44.1 kHz sampling rate

### 🌐 SRT Protocol Integration
- Supports **SRT streaming** to remote server  
- Configurable endpoint (IP & Port)  
- Caller mode support

## 🛠 Technologies Used

- **Mobile Platform**: Android (Java/Kotlin)  
- **Streaming Library**: StreamPack  
- **Video Codec**: H.264 (AVC)  
- **Audio Codec**: AAC  
- **Protocol**: SRT (Secure Reliable Transport)  
- **Networking**: Local Network / IP-based Streaming  

---

## 🔄 System Workflow

1. User opens the Android app  
2. Configure streaming parameters:
   - Resolution  
   - Bitrate  
   - FPS  
   - SRT Server (IP & Port)  
3. Connect to SRT server  
4. Start live streaming  
5. Video & audio encoded in real-time  
6. Stream transmitted to server  
7. Server receives and displays stream
