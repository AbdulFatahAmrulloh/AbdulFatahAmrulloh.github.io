This project presents a full-stack internal website developed to support **Technical Operation Support** teams by providing real-time access to crew shift schedules and operational information. The system is designed to centralize scheduling data, improve coordination, and ensure accurate information flow across departments.

## Key Features

- **Real-Time Shift Schedule Display**:  
  Displays daily and monthly crew shift schedules dynamically, ensuring all teams can access the most up-to-date information.

- **Excel-Based Data Management**:  
  Supports Excel file uploads for automatic updates of crew schedules and contact information, reducing manual data entry and errors.

- **Crew Information by Active Shift**:  
  Automatically displays crew names, roles, and contact numbers based on the currently active shift.

- **Cross-Team Data Integration**:  
  Ensures seamless data synchronization between production, COGF, and technical support teams to enhance operational efficiency.

- **Responsive Web Interface**:  
  Optimized for different screen sizes to allow access from control rooms, offices, and operational environments.

## Technologies Used

- **Backend**: PHP (server-side logic), Python (data processing and Excel parsing)
- **Frontend**: HTML, CSS, JavaScript
- **Database**: MySQL (crew data and shift schedules)
- **Design & Prototyping**: Figma (UI/UX design)
- **Data Source**: Excel (.xlsx) for schedule and contact updates

## Outcome

This system aims to:
- Centralize technical crew scheduling in a single internal platform  
- Reduce miscommunication related to shift assignments and crew availability  
- Improve operational coordination through real-time and accurate schedule data  
- Serve as a scalable foundation for future operational support tools
